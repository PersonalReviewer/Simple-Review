# Security Policy

## Supported versions

This project is early. Security fixes will target the current `main` branch until release branches exist.

## Reporting issues

Open a GitHub issue for normal bugs.

For sensitive security reports, contact the maintainer privately if a contact method is listed on the GitHub profile.

## Token safety

Never paste GitHub tokens, API keys, credentials, or private review data into issues, commits, screenshots, or support chats.

If a token is exposed, revoke it immediately and create a new one.

## Local data

Review Studio stores review files locally as JSON. Users are responsible for protecting their own local data directory and exports.
